# Moirai — Frontend Redesign Handoff

This folder is the design handoff package for **Moirai** (site: **moirai.gg**), a UFC fight prediction system. It's intended to be uploaded to **Claude Design** (claude.ai/design) so it can generate a polished frontend redesign without producing the generic dark-dashboard default.

## What the project is

Moirai is a Python-based UFC fight prediction system. A LightGBM ensemble (with isotonic calibration and split-conformal intervals) produces, for every bout on every upcoming UFC card:

- A calibrated win probability for each fighter, plus a 90% confidence interval
- A predicted method breakdown (KO/TKO, Submission, Decision) for each fighter
- A round-by-round finish distribution (R1–R5)
- An "edge" analysis comparing the model to live SportsBet odds, with empirically-gated value-bet flags and Kelly sizing
- A retrospective scorecard for past events (predicted vs. actual)

The model is rebuilt weekly and predictions refresh daily via GitHub Actions. The frontend is the model's primary output surface — what the user opens to decide what to bet on for the upcoming card.

## Current frontend stack

- **No JS framework.** The frontend is a single static HTML page, `docs/index.html`, generated by `ufc_predict/serve/build_dashboard.py` via Jinja2 from `ufc_predict/serve/templates/dashboard.html` (~2,000 LOC of inline HTML/CSS/JS).
- Inputs: `data/predictions.json`, `data/past_events.json`, `data/upcoming_schedule.json`, `data/fighter_metadata.json`, `data/fighter_images.json`, `data/model_performance.json`.
- Hosted on GitHub Pages, branch deploy from `/docs`.
- Current visual identity: previously "SharpOctagon", more recently "Apex" — dark background with red/blue corner colours, Matrix-green accent, gradient header, terminal-style typography. We're replacing this aesthetic and rebranding to **Moirai** (the site lives at **moirai.gg**).
- The redesign should keep the static-site delivery model in mind: anything generated will be re-templated into Jinja2 and emitted as a single HTML file, but Claude Design can output React/Next/whatever it wants for the prototype — the user will port it.

## Screens to redesign

The current dashboard has three top-level tabs plus an inline-expanding drill-down. Map them as four screens:

1. **Upcoming Fights** (priority: must-have, design first) — home screen. Opens with a **slow horizontally auto-scrolling hero bar** of featured bout cards (premium ticker feel), then a dense list of upcoming events: each with its bouts, win probabilities, predicted method, key drivers.
2. **Fight Detail** — drill-down view for a single bout: fighters side-by-side, full prop breakdown (method, round, distance), feature drivers, prediction confidence interval, and the bet card with edge vs. SportsBet odds.
3. **Top Bets** — portfolio view across all upcoming events: Kelly-sized recommended bets, per-event bankroll allocation, filterable by bet type. Has empirical-ROI gating (the model only flags markets that have shown profit in backtests).
4. **Past Events** — completed events with predicted-vs-actual scorecards, highlighting which model picks hit and which missed; rolling accuracy / Brier / log-loss metrics.

## Notes / assumptions flagged

- The current dashboard renders flag emojis next to fighter names. The redesign explicitly drops these (see anti-patterns in `design-brief.md`). The data still carries `fighter_a_nationality` / `fighter_b_nationality` if a more tasteful nationality treatment is wanted.
- "Method" in props is split per fighter (`prob_a_wins_ko_tko`, etc.) and also as fight-level neutral aggregates (`prob_finish`, `prob_decision`). Either can be the headline.
- Some bouts have `prob_a_wins == 0.5` exactly — that's a sentinel meaning the model didn't run for that bout (early prelims often). The design needs an "ungraded / preview only" state for these, not a 50/50 chart.
